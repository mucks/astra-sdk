Architecture
Astra SDK architecture

In this diagram, the arrows represent data flow

Astra consists of several libraries. The main library is astra_core.dll. (Or libastra_core.so on Linux-style systems.) Astra_core.dll provides support for delivering frames via StreamReaders and loading plugins that provide Streams and StreamSets. Astra_core.dll delivers frame data but it does not know what is inside the frame. It treats them as opaque byte arrays. This gives us the flexibility to add new streams in the future without changing the API or astra_core.dll. To add new stream types, such as new middleware, we can update a plugin or add a plugin.

The second library is astra.dll. This library knows about the plugins that ship with Astra SDK, so it can interpret the frame byte arrays from astra_core.dll into concrete data types for the Streams. It also provides mechanisms to control and configure plugins, such as changing the resolution or checking the field of view. The application that uses Astra SDK uses both astra.dll and astra_core.dll when it uses the API. Plugins also use astra.dll to understand the frame data provided from other plugins.

Astra SDK ships with several plugins. Together, these plugins provide the data sources for all of the Streams in Astra SDK. The plugin openni_sensor.dll uses OpenNI2 as a backend for accessing raw sensor data, including color and depth. Other plugins contain middleware that consume the raw sensor Streams and publish derivative Streams such as PointFrame and BodyFrame.

All of this happens behind the scenes and the SDK only computes what is needed to provide the host application with the streams that it requests. For example, if an application does not start the BodyStream, then body tracking is not calculated.

There is also another library, astra_core_api.dll which serves as a proxy for the Astra API and forwards all API calls back to astra_core.dll. It is needed for plugins for technical reasons but it is not important for applications developers to know about. As long as all of the libraries in Astra SDK are present, then the system will work.
